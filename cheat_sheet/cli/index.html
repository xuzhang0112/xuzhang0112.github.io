常用软件#

VIM#

vim filename # 我只会这一条

  1. 正常模式（Normal-mode)

    • 狂按进入
    • dd删除一行，p粘贴

  2. 插入模式（Insert-mode)

    • 按 i 键会进行插入模式，进入编辑状态，通过键盘输入内容。

  3. 命令模式（Command-mode)

    • 先切正常模式，然后按：或/ 进入
    • 未编辑，按q退出；编辑后，wq保存退出，q!不保存强制退出

CUDA#

CUDA主要有两个API：runtime API、driver API

Driver version：可通过nvidia-smi查看显卡的驱动和CUDA驱动版本，CUDA驱动版本需要高于CUDA的runtime版本

Runtime version: 可通过nvcc -V查看，表示CUDA Toolkit的runtime版本，通过CUDA Toolkit installer安装，见usr/local/cuda/include/cuda.h），需要高于pytorch编译时的CUDA版本

Conda安装的CUDA toolkit是不完整的，而且当系统里存在CUDA toolkit时，会优先调用系统级的CUDA

tensorflow和pytorch可以直接调用服务器预装的CUDA和cuDNN (pytorch不需要cuDNN)

Git#

本人git小白，在此记录自己使用的一些工作流（低情商：只会这么多）

初始化，一般是本地或者克隆他人的仓库

git init # 创建git仓库
git clone [url] # 复制仓库  
touch .gitignore # 修改.gitignore文件在其中写入需要忽略的文件和目录
#“/”表示从.gitignore所在的根目录开始匹配末尾的"/"表示匹配目录而不是文件

本地

git add . && git commit -m 'update'

远程仓库

git remote add origin [https://username:password@github.com/repo] #注意私有仓库的密码是你的token
git push -u origin main #第一次需要指定upstream
git add . && git commit -m 'update' && git push # 远程仓库一条龙

合并

git add . && git commit #首先确保本地的修改已经提交
git pull # 然后拉取远程仓库中的修改
git config pull.rebase true # 可以配置一下，比较省事
git add . && git commit -m 'update' && git push # 对修改过目之后，即可远程仓库一条龙，进行覆盖
