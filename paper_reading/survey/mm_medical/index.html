Dataset

MIMIC-CXR
#

377,110 images, 227,835 reports

NIH-CXR
#

112,120 X-ray images from 30,805 unique patients, with labels of 14 diseases

The labels are expected to be >90% accurate and suitable for weakly-supervised learning

CheXpert
#

224,316 images of 65,240 patients, labels of 14 diseases: positive, negative, uncertain

Train: all labeled by CheXpert Labeler

Validation: 200, labeled by experts

Test: 500, labeled by experts

PadChest
#

  • 160,000+ images
    • from 67,000 patients, from 2009 to 2017
  • reports
    • labeled with 174 different radiographic findings, 19 differential diagnoses and 104 anatomic locations
    • 27% were manually annotated and the remaining set was labeled by RNN (0.93 Micro-F1 score)

RadGraph
#

structure clinical information in a radiology report with 4 entities and 3 relations, with MIMIC-CXR and CheXpert

https://physionet.org/content/radgraph/1.0.0/

  • Annotations: 500 for train, 100 for test
    • train.json: File containing annotations obtained by board-certified radiologists for 425 radiology reports (MIMIC-CXR)
    • dev.json: File containing annotations obtained by board-certified radiologists for 75 radiology reports (MIMIC-CXR)
    • test.json: File containing annotations obtained by 2 different board-certified radiologists for 100 radiology reports (50 MIMIC-CXR, 50 CheXpert)
  • Graphs
    • MIMIC-CXR_graphs.json: File containing annotations obtained by our deep learning model for 220,763 radiology reports (MIMIC-CXR)
    • CheXpert_graphs.json: File containing annotations obtained by our deep learning model for 500 radiology reports (CheXpert)
  • Models
    • models/model_checkpoint: Folder containing the saved model parameters used to automatically generate the annotations in MIMIC-CXR_graphs.json and CheXpert_graphs.json
    • models/README.txt: File containing instructions for performing inference using model checkpoint
    • models/inference.py: File containing code to extract entities and relations given a directory containing reports in txt format

Contrast-Attentive Thoracic Disease Recognition With Dual-Weighting Graph Reasoning [TMI 2021]
#

  • Intra- and inter-contrastive abnormal attention

  1. Intra-contrastive

医学先验：某些肺病通常表现为左右肺的不对称

先使用DenseNet121提取左右肺特征，做差后，GMP，得到全局特征f_intra

  1. Inter-Contrastive

先使用聚类算法找出3个最具有代表性的normal CXR作为anchor scan，然后通过注意力机制对query scan中相似的部分进行增强，得到3个增强后的特征，然后通过矩阵变换，得到最后的融合全局特征f_inter

  • Dual-weighting graph reasoning

Nodes: N=43, text embedding is obtained by BioSentVec trained on PubMed and MIMIC-III

Edges: base on coocurance matrix

Dual Weighting: 通过双权重的双层GNN来使43个节点的概念充分交互，捕捉节点之间的相互依赖关系

  • Multi-Disease Recognition
  1. Visual Signals
    1. f_global: 特征提取得到，并经过注意力增强
    2. Graph Node Vector: 细粒度的特征，soft probability of 43 nodes predicted based on image feature
    3. f_visual: f_global和graph node vector的外积，表示每个像素处43个节点的概率
  2. Graph Signals
    1. Graph embedding: 预先构建的knowledge graph，并通过GNN进行优化，节点的embedding表示相应的医学概念
    2. including findings, diseases or syndromes, qualitative concepts, spatial concepts, temporal concepts, and body locations or regions
  3. Prediction
    2. Disease Identification
    3. Disease Localization
      1. MIL: 在进行像素级训练时，将所有像素的类别相乘，作为图片级别的类别，然后使用图像级别的标注进行优化
  4. Training Strategy

先训练特征提取器（多标签分类），然后训练图卷积和视觉卷积

AnaXNet: Anatomy Aware Multi-label Finding Classification in Chest X-ray [MICCAI 2021]
#

  • 代码：https://github.com/Nkechinyere-Agu/AnaXNet
  • Faster-RCNN：提取解剖ROIs以及相对应的特征，即k个解剖结构的特征，维度为d

  • Anatomy GCN
    • 基于co-occurance获取描述anatomy之间关系的邻接矩阵，并通过设定阈值滤除噪声，捕捉解剖结构之间的依赖关系
    • 基于依赖关系，聚合各个解剖结构的特征
  • Cross Anatomy Attention
    • 基于相似度，进一步整合所有解剖结构的特征，并使用残差连接保持原始特征
  • FC：对于每一个解剖结构的embedding使用分类头进行疾病预测

AGXNet: Anatomy-Guided Weakly-Supervised Abnormality Localization in Chest X-rays [MICCAI 2022]
#

  • 代码：https://github.com/batmanlab/AGXNet

  • 细粒度的监督
    • 对Report进行解析，得到Anatomy和Observation的邻接矩阵，第i行第j列表示第i个解剖结构上是/否/不确定存在第j种异常，其中的正例和负例相当于同时给解剖结构和异常观察提供了0/1的标签
    • 使用BCE Loss对Anatomy Network和Observation Network进行多标签分类的训练，训练其判断每个类别的anatomy/observation异常是否存在，并将Anatomy Network的feature map作为注意力图辅助Observation Network进行训练
  • PU Learning
    • Best Bin Estimation (BBE) 是一种用于估计未标记训练数据中正例占比的方法。
    • BBE 基于一个假设，即未标记数据的正例和负例分别具有不同的概率分布，但是这些分布的某些区域是相互重叠的。因此，BBE 的目标是找到这些重叠区域的最佳二进制划分，从而最小化正例占比的估计误差。
    • 我们重复步骤 ，直到分类器达到最佳验证性能

Spatial-Hierarchical Graph Neural Network with Dynamic Structure Learning for Histological Image Classification [MICCAI 2022]
#

  • 代码：https://github.com/HeLongHuang/SHGNN
  • 空间多尺度特征提取

对同一张图像处理得到不同尺度的Patch，使用预训练的ResNet34对图像特征进行编码，得到空间多尺度特征

  • 图拓扑结构的动态学习

将视觉特征和位置坐标共同编码得到每个entity的联合特征，然后使用K近邻算法将每个节点与它的k个最相似节点建立边的连接（这些节点必须小于阈值），作为图的拓扑结构，即邻接矩阵

  • 空间多尺度交互

使用GNN在L个不同的尺度对特征进行聚合，然后沿着将每个细胞和它所属的所有尺度组织拼接成token，使用VIT进行全局的信息融合，再使用GNN按照细胞尺度的拓扑结构进行聚合，最后得到分类特征图

  • 分类

使用全局平均池化将特征拓扑图转化为特征向量，再使用MLP进行分类

Dynamic Graph Enhanced Contrastive Learning for Chest X-ray Report Generation [CVPR 2023]
#

代码：https://github.com/mlii0117/DCL

  • Overview
    • A dynamic graph to construct the relationships between diseases and organs
    • Represent Image with medical knowledge graph embeddings with Cross Attention
  • Dynamic Graph
    • $$G_{pre}$$: 27 nodes, 20 disease keywords, 7 organs or tissues
    • $$G_{specific}$$: retrieve top-N reports, extract keywords using Stanza, retrieve knowledge in RadGraph, add nodes to $$G_{pre}$$,
    • Embedding: use SciBert to get the initial embedding, add level embedding to show which level the node is at (disease, organ, tissue and so on)
  • Learning
    • IRC：Report embedding& Image(Graph) embedding -> Contrastive Learning
    • IRM: whether image and report match,
      • Q: Report
      • K, V : Image
      • CE Loss
    • RG: Self Regression, Teacher Forcing

KiUT: Knowledge-injected U-Transformer for Radiology Report Generation [CVPR 2023]
#

  • Signals

    • Visual Knowledge signal
      • ResNet101 -> 分patch -> RRSA
      • RRSA (Region Relationship Self Attention) : 不仅考虑patch特征的相似度，还考虑两个patch之间的相对位置
    • Clinical Knowledge Signal
      • symptom node embedding (from MBERT) ✖️ symptom probability（from classifier）
      • Graph Attention: based on symptom relationship graph
    • Contextual Knowledge Signal
      • Previous output -> MBERT -> Masked MHA

  • U connection

    • 自回归式Decoder，考虑previous output和encoder visual signals
    • Cross Attention的Q是previous output，但K和V不再是last Encoder的输出，而是N-i+1个Encoder的输出

  • Injected Knowledge Distiller

    • Last Decoder的输出通过attention整合clinical knowledge signal和contextual knowledge signal，沿序列维度拼接后预测下一个词的输出

    • K-Diag: Knowledge-enhanced Disease Diagnosis in Radiographic Imaging [MICCAI 2023]

    • https://github.com/MediaBrain-SJTU/K-Diag

    • Learnable Prompt Module

    • 将Q种疾病用N种特征来表达（特征是可学习参数）

    • 疾病Qd 特征Nd 疾病在特征上的概率分布通过MLP得到QN 最终表达Qd

    • Disease-Query Module

    • Architecture: a 4-layer transformer decoder with an MLP to get the final prediction

    • queries: disease embeddings (by pre-trained knowledge encoder and learnable prompt module)

    • keys&values: the encoded features

    • binary classification, CE loss

    • MedKLIP: Medical Knowledge Enhanced Language-Image Pre-Training [ICCV 2023]

    • https://github.com/MediaBrain-SJTU/MedKLIP

    • pipeline

      • Image embedding与某种疾病的entity knowledge embedding(先通过wiki&UMLS知识库构建,然后通过BoiClinical-Bert编码)共同输入模型，预测这种疾病的有/无，位置，特征图

      • 训练时，选取Train Set中的Top Q疾病的知识编码进行训练；
      • 推断时，基于该疾病的医学知识编码进行推断

    • 训练

      • Pseudo label
        1. 疾病是否存在
        2. 疾病处于何处：使用BioClinical-Bert进行编码
      • Fine-grained supervision
      • 疾病有无：BCE Loss
      • 疾病位置：将该疾病的位置编码和其他M种位置编码进行对比学习（因为位置编码的类别数未知）

    • KAD: Knowledge-enhanced Visual-Language Pre-training on Chest Radiology Images [Nature Communication]

    • https://github.com/xiaoman-zhang/KAD

    • Knowledge database

      1. knowledge graph: concept-relation-concept triplets
      2. concept info list: concept-definition pairs

    • Pre-training

      1. Text contrastive learning between medical concepts or definitions
      2. image-text contrastive learning with paired chest X-rays and extracted entities(by tools like ChatGPT)
      3. optimize a Disease Query Network (DQN) for classification

    • Zero-shot inference

      1. encode the disease name as a query input, and DQN will output the probability that the pathology is present in the input image

Towards Generalist Foundation Model for Radiology
#

https://github.com/chaoyi-wu/RadFM

  • Dataset
    • Medical Multimodal Dataset (MedMD)
      • visual-language interleaved, 2D&3D, research-oriented &real world clinical data
      • subset
        • Interleaved Dataset (CLIP)
          • no strong question-and-answer relationships between contexts
          • Eg. PMC-Inline(将插图定位到论文中的具体某一段落文字)
        • Visual Instruction Datasets (Q&A)
          • eg. “What can you see from the image? I can see lesions.”
          • 包括yes or no判断和开放式问答
    • Radiology Multimodal Dataset (RadMD)
      • a filtered radiology subset (对不同的子数据集采取不同的过滤策略)
      • contains a total of 3M images, spanning various data formats, modalities, and tasks
  • Unified Learning Paradigm

  • Architecture
    • Visual encoding
      • convert the image into 3D patches, and embed them into a token sequence and feed it into the encoder
        • $$v_i=\phi_{vis}(v_i)\in R^{P_i*d}$$
        • adopt 3D ViT here to be compatible with both 2D and 3D image input
        • the input image sizes to be divisible by the patch size of 32 × 32 × 4, so we resize the depth dimension to the nearest multiple of 4 and will not surpass 64
          • Eg. 2D input 512512 -> 512512*4
        • $$P_$$ is the total number of tokens. Due to the inconsistency in depth dimension, $$P_$$ also varies across 2D and 3D images. Generally, 3D cases may have much larger $$P_$$ than 2D cases
    • Aggregation with Perceiver
      • After visual encoding, adopt a perceiver module $$\phi_{per}$$ to aggregate visual representation
        • $$\phi_{per}$$ is a transformer decoder with a fixed number of learnable queries
        • visual input is treated as key and value
        • final output embeddings will be normalized into the same dimension with the pre-defined learnable query sequence
    • Multi-modal fusion
      • interleave the visual embedding with text embeddings from tokenization, where the special image placeholder token is simply replaced with the corresponding visual embedding.
      • The interleaved sequence is then passed into a decoder-only large language model (Φllm) to obtain the output
  • Loss function
    • $$\omega_l$$: medical-related单词具有更高的损失权重（属于UMLS)，非医学相关的word具有更低的权重，image placeholder token没有权重
